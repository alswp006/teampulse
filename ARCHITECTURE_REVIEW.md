# Architecture Review
Generated: 2026-07-21T19:19:48.439Z
Score: 68/100

## Summary
The codebase follows a fairly clean page/component/lib layering with good documentation and no circular dependencies, but it suffers from repeated boilerplate (cache-unwrapping, haptic-firing, and data-fetching effect logic copy-pasted 4x each) and a non-trivial amount of dead code (unused components, unused utils, and vestigial value exports paired with types). One notable consistency issue: a purpose-built `TossRewardAd` wrapper exists but `Report.tsx` reimplements the same ad-gating flow directly against the SDK instead of using it. Overall the architecture is sound but needs a deduplication/cleanup pass.
