# @apps-in-toss/web-framework — 실제 SDK API 레퍼런스

이 파일은 실제 설치된 `@apps-in-toss/web-framework@2.4.x`의 `.d.ts`에서 검증된 API만 기술합니다.
여기에 없는 API는 SDK에 없습니다. 상상하지 말 것.

패키지 구조:
- `@apps-in-toss/web-framework` = `@apps-in-toss/web-bridge` + `@apps-in-toss/web-analytics` re-export
- 실제 import 경로는 `@apps-in-toss/web-framework` 그대로 사용

```typescript
import { Storage, Analytics, grantPromotionReward } from '@apps-in-toss/web-framework';
```

---

## 절대 원칙

1. **React 훅(`use*`)은 SDK에 없다.** 템플릿이 제공하는 `useTossLogin`, `useTossAd`, `useTossPayment`는 프로젝트 로컬 래퍼이며, 이 가이드에서 권장하지 않음. 직접 imperative API를 호출하라.
2. **광고, 결제, 위치 등은 imperative 함수 + 이벤트 콜백 구조**이다. Promise-only가 아닌 경우가 많음.
3. **로그인 훅은 없다.** 토스 앱 자체가 사용자 세션을 제공하므로 별도 login 호출 불필요. 통합 여부만 `getIsTossLoginIntegratedService`로 확인.
4. **분석/로깅은 `Analytics` 사용.** 외부 툴(GA, Amplitude) 금지.

## 1. 저장소 (Storage)

네이티브 저장소. localStorage보다 persistence 보장이 강함.

```typescript
import { Storage } from '@apps-in-toss/web-framework';

await Storage.setItem('my-key', 'value');           // 저장
const value = await Storage.getItem('my-key');       // 조회: string | null
await Storage.removeItem('my-key');                  // 삭제
await Storage.clearItems();                          // 전체 초기화
```

**주의**: value는 `string`만. 객체는 `JSON.stringify` 필요.

## 2. 인앱결제 (IAP)

### 2.1 1회성 결제

```typescript
// ⚠️ IAP는 최상위 export가 아니라 IAP 네임스페이스 아래에 있다(.d.ts 검증).
//    import { createOneTimePurchaseOrder } ... → 존재하지 않음(빌드 에러).
import { IAP } from '@apps-in-toss/web-framework';

// 반환값은 cleanup 함수 → 언마운트/완료 시 반드시 호출해 리소스 해제
const cleanup = IAP.createOneTimePurchaseOrder({
  options: {
    sku: 'product-id',                    // 앱인토스 콘솔에 등록한 SKU
    processProductGrant: async ({ orderId }) => {
      // 백엔드에 orderId 전송 → 상품 지급 처리
      // 반환: true (성공) | false (실패)
      return true;
    },
  },
  onEvent: (event) => {
    // event.type === 'success', event.data = IapCreateOneTimePurchaseOrderResult
    console.log(event.data.orderId, event.data.amount);
  },
  onError: (error) => {
    console.error(error);
  },
});
// cleanup();  // 결제 흐름 종료 시

// React 앱은 이 imperative API를 직접 쓰지 말고 템플릿의 src/components/TossPurchase.tsx
// (WebView-밖 throw 가드 + cleanup 관리 래퍼)를 사용하는 것을 권장.
```

**IapCreateOneTimePurchaseOrderResult 필드**:
- `orderId`, `displayName`, `displayAmount` (예: "1,000원"), `amount` (숫자), `currency` (ISO 4217), `fraction` (소수점 자리수), `miniAppIconUrl`

### 2.2 구독 결제

```typescript
import { IAP } from '@apps-in-toss/web-framework';

const cleanup = IAP.createSubscriptionPurchaseOrder({
  options: {
    sku: 'subscription-id',
    offerId: 'offer-free-trial',    // optional — null이면 기본 가격
    processProductGrant: async ({ orderId, subscriptionId }) => true,
  },
  onEvent: (event) => { /* SubscriptionSuccessEvent (event.data) */ },
  onError: (error) => { /* ... */ },
});
// cleanup();  // 종료 시
```

### 2.3 Offer 타입

- `'FREE_TRIAL'` — 무료 체험 제공
- `'NEW_SUBSCRIPTION'` — 신규 구독자 할인
- `'RETURNING'` — 복귀자 할인

## 3. 광고 (Ad)

SDK는 **두 종류**의 광고 API를 제공. React 컴포넌트가 아닌 **imperative 함수**.

### 3.1 배너 광고 (TossAds)

```typescript
import { TossAds } from '@apps-in-toss/web-framework';

// 1) 초기화 (앱 라이프사이클당 한 번)
if (TossAds.initialize.isSupported()) {
  TossAds.initialize({});  // InitializeOptions: { callbacks?: { onInitialized, onInitializationFailed } }
}

// 2) DOM 노드에 배너 부착 — adGroupId는 앱인토스 콘솔 발급
const targetEl = document.getElementById('ad-slot');
if (TossAds.attachBanner.isSupported() && targetEl) {
  const result = TossAds.attachBanner('banner-ad-group-id', targetEl, {
    theme: 'auto',          // 'auto' | 'light' | 'dark'
    variant: 'card',        // 'card' | 'expanded'
    tone: 'blackAndWhite',  // 'blackAndWhite' | 'grey'
    callbacks: {
      onAdRendered: (p) => { /* p: { slotId, adGroupId, adMetadata } */ },
      onAdClicked: (p) => { /* ... */ },
      onAdFailedToRender: (p) => { /* p.error: { code, message } */ },
      onNoFill: (p) => { /* ... */ },
    },
  });

  // 3) 컴포넌트 언마운트 시
  // result.destroy();
}

// (legacy) loadAdMob / showAdMob / isAdMobLoaded 는 SDK에 존재하지 않음 — 사용 금지
```

### 3.2 전면 광고 / 리워드 광고 (FullScreen Ad)

```typescript
import { loadFullScreenAd, showFullScreenAd } from '@apps-in-toss/web-framework';

// 로드
loadFullScreenAd({
  slotId: 'reward-slot-id',
  onEvent: (event) => { /* LoadFullScreenAdEvent — loaded 등 */ },
  onError: (error) => { /* ... */ },
});

// 표시 (시청 완료 시 리워드 지급 기준점)
showFullScreenAd({
  slotId: 'reward-slot-id',
  onEvent: (event) => {
    // event.type이 'rewarded' | 'dismissed' 등으로 구분됨
    // 시청 완료 후 비즈니스 로직 실행
  },
  onError: (error) => { /* ... */ },
});
```

**주의 사항**:
- 광고 API는 **앱인토스 WebView 런타임에서만 동작**. 로컬 브라우저 개발 시 `onError`로 떨어질 수 있음 → fallback 필요
- React 래퍼(`TossRewardAd`, `AdSlot`)는 **SDK에 없음**. 프로젝트가 필요하면 위 imperative API를 감싸서 직접 만들어야 함

## 4. 포인트 지급 (Promotion)

```typescript
import { grantPromotionReward } from '@apps-in-toss/web-framework';

await grantPromotionReward({
  promotionCode: 'CONSOLE_ISSUED_CODE',   // 앱인토스 콘솔에서 발급
  amount: 1000,                            // 1인당 최대 5,000원 — 초과 시 호출 금지
});
```

**정책**:
- 1인당 누적 5,000원 한도 (amount > 5000 호출 금지)
- promotionCode는 반드시 앱인토스 콘솔에서 발급받은 것 사용
- 용도: 첫 사용 보상, 친구 초대 리워드, 이벤트 참여 보상

## 5. 로그인 상태 확인

**SDK에 로그인 API는 없다.** 토스 앱이 이미 사용자 세션을 제공하므로 별도 login 불필요.
앱인토스 미니앱은 기본적으로 **익명 또는 토스 통합 세션** 중 하나.

```typescript
import { getIsTossLoginIntegratedService } from '@apps-in-toss/web-framework';

const isIntegrated = await getIsTossLoginIntegratedService();
// true: 토스 로그인 통합됨, false: 익명 서비스
```

사용자 식별이 필요한 서비스는 앱인토스 콘솔에서 "토스 로그인 통합"을 설정해야 함.

## 7. 분석 / 로깅 (Analytics)

외부 로깅 툴(GA, Amplitude) 사용 금지. 반드시 SDK의 Analytics 사용.

```typescript
import { Analytics } from '@apps-in-toss/web-framework';

// 화면 진입
Analytics.screen({ log_name: 'HomeScreen', user_id: 'xxx' });

// 노출 (impression)
Analytics.impression({ log_name: 'HomeBanner' });

// 클릭
Analytics.click({ log_name: 'CalculateButton', salary: 50000000 });
```

**params 타입**: `{ log_name?: string } & { [key: string]: Primitive }`
(Primitive = string | number | boolean | null)

## SDK에 "없는 것" (쓰려고 하지 말 것)

❌ `useTossLogin`, `useTossAd`, `useTossPayment` — React 훅 형태 API 일체 없음
❌ `TossRewardAd`, `AdSlot` — React 컴포넌트 형태 광고 래퍼 없음 (직접 만들어야 함)
❌ `window.Toss.xxx` 글로벌 — 구식 WebView 브릿지 패턴, 현재 SDK에서 사용 금지

---

## 작성 원칙 (spec/task/coding 에이전트에게)

1. 이 파일에 있는 API만 사용한다고 가정하고 스펙 작성
2. React 래퍼가 필요한 경우 "프로젝트 로컬 컴포넌트로 구현" 명시
3. 광고 시나리오는 `loadFullScreenAd` + `showFullScreenAd` 조합, Promise 아님
4. 결제 시나리오는 `createOneTimePurchaseOrder` + `processProductGrant` 콜백 구조
5. 외부 분석 툴 언급 금지 — Analytics로만
6. "이 API 없음" 정보가 중요할 때는 위 "없는 것" 목록 참조
